Key projects

Festival of Bright Ideas

URL: https://festivalofbrightideas.com.au/
Client: Luminous Events

The design for this website included multi leveled parallax scrolling for the home page. There's three levels working at different speeds, the 'clouds' (blue and pink), the white stars and then the images and text. I couldn't find an off-the-shelf package that did exactly what I needed so I ended up building it from scracth using js. I'm proud of how this project turned out because I was working closely with the designer on the implementation, going back and forth on the scrolling speed, the posistioning of all the different elements and the layout so it worked on all screen sizes. I think together we created a great user experience without sacrificing performance. The custom nature of the solution meant I could change it on the fly to meet any tweaks in the design.

Earthling Cup

URL: https://earthlingcup.com
Client: Blue Chapel

Using the Shopify API I was able to create a bespoke frontend not possible using the standard Shopify site builder. All interactions including cart management, inventory levels and stock info is handled via the API. All information about stock is entered via Wagtail (the CMS) and then sent to Shopify so the client doesn't need to ever use Shopify directly. Shopify can then be leveraged when needed to for things like shipping calculations and payment processing.

Digital Ready Checkup

URL: https://checkup.digitalready.tas.gov.au/
Client: Department of State Growth

This project was a good chance to flex my skills across frontend, backend and devops. The whole app had to work asynchronous so it could deal with with constant user input from the 'chat' while the backend was working in the background with the long running proccesses used to analyze the websites. To achieve this, websockets were used to send info back and forth between the frontend, written in React, and the backend, written in Python and using Tornado as a framework. AWS lambda was used to offload the long running processes e.g. we were able to package a slimmed down Chrome binary that ran Google lighthouse against a url, all within a standard Node lambda environment. I was really stoked with the end product and since launch we've been able to add in more website analysis tests and extend and enhance the frontend with minimal friction because of how the app was architected.

Infrastructure Overhaul

Client: Neon Jungle

When I started at Neon Jungle deployments were done using shell scripts, manually triggered by a developer, pushed out to a vps that nobody was quite sure where it was. Since taking over the infrastucture, all sites and apps are now containerised using Docker. Logging is pushed out to an external service and Bugsnag was added to automatically notify of any production errors. Building and deploying of containers is all automated using Gitlab pipelines, so that any push to a master branch for a site runs the test suite and deploys on success. This means the master branch can be the source of truth and no developer can deploy the wrong code, or overwrite another developer's deployment. All ~40 sites once containerised were able to fit on a single micro ec2 instance, with routing and SSL handled by Traefik, significantly reducing hosting costs.
